# PM Agent — Charter Best Practices

## High-Quality Charter Checklist
A charter scores > 0.90 confidence when it includes ALL of the following:

- [ ] Project title, client name, and assigned PM
- [ ] Clear objective (one sentence, outcome-focused)
- [ ] Timeline in weeks with milestone markers
- [ ] Budget with contingency (typically 10–15%)
- [ ] At least 3 success criteria that are measurable
- [ ] Work Breakdown Structure (WBS) with at least 4 phases
- [ ] Risk register: minimum 3 risks, each with likelihood + mitigation
- [ ] Stakeholder communication plan (who gets what, how often)
- [ ] Approval gate: Trice Johnson (CDO) must sign off before kickoff

## WBS Standard Phases (use for all projects)
1. Discovery & Requirements (weeks 1–2)
2. Design & Architecture (weeks 3–4)
3. Build & Integration (weeks 5–9)
4. QA & Testing (weeks 10–11)
5. Delivery & Handoff (week 12)

## Risk Categories to Always Address
- **Technical risk** — platform complexity, integration unknowns
- **Resource risk** — key person dependency, vendor reliability
- **Scope risk** — client expectation drift, undocumented change requests
- **Timeline risk** — external dependencies (e.g., client approval delays)
- **Budget risk** — vendor cost overruns, scope creep cost impact

## Lessons Learned — Charter Quality
- **LESSON-001:** When BA Agent reviews the design session before PM runs,
  charter quality improves ~15%. Sequence: BA first, then PM.
- **LESSON-002:** Charters with fewer than 3 risks are rejected at the approval
  gate 80% of the time. Always include at least 5 risks.
- **LESSON-003:** Including a "Definition of Done" section reduces scope disputes
  post-delivery. Add it to every charter.
- **LESSON-004:** AURA MVP timeline is locked at 3 months. Never deviate.
